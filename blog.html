使用 AI Agent 辅助搭建个人主页：一次算法基础大作业实践

这篇博客记录我如何借助 AI Agent 完成算法基础课程个人主页，从需求理解、页面设计到实现调试，也整理了我对 AI 辅助编程的一些真实感受。

1. 为什么要搭建个人主页

在学习算法基础这门课之前，我对“个人主页”的理解比较简单：它可能只是一个展示姓名、项目和联系方式的网页。但真正开始做这个作业之后，我发现个人主页也可以成为一个学习整理工具。

算法学习中有很多概念并不是看一遍就能真正掌握，例如复杂度分析、动态规划状态设计、图算法中的遍历方式等。如果只是把笔记放在本地文件夹里，时间久了很容易散掉。把这些内容整理到网页上，会迫使我用更清楚的结构表达自己的理解。

个人主页不仅是展示结果的地方，也可以是记录过程、训练表达和沉淀知识的空间。

2. 使用的工具与技术

这次项目采用静态网页实现，重点不是追求复杂框架，而是让页面结构清晰、内容可访问、后续容易修改。

  • AI Agent：用于辅助分析需求、生成页面结构、优化样式和检查交互细节。
  • HTML / CSS / JavaScript：用于实现首页、博客页、响应式布局和基础动态效果。
  • Git：适合后续进行版本管理，记录每一次修改过程。
  • Markdown：适合先组织博客内容，再转换成网页正文。
  • GitHub Pages / Vercel / Netlify：可以用于部署静态网站，让主页能够被正常访问。

3. 实现过程

实现时我先把课程要求拆成两个核心目标：第一，主页本身要像一个完整的个人作品；第二，主页必须能明确跳转到第一篇技术博客。围绕这两个目标，我把首页分成 Hero、About、Projects、Learning Notes、Blog Preview 和 Contact 几个区域。

确定结构后，再统一设计蓝色科技感的视觉风格。主页背景使用浅蓝渐变、细网格和柔和光斑，卡片采用半透明玻璃效果，按钮和标签使用蓝色渐变。这样可以保持计算机方向的科技感，同时避免过度花哨。

最后添加博客入口、测试页面跳转，并检查移动端布局。相比一开始直接写代码，先拆需求再实现会更稳，也更容易判断页面是否满足作业要求。

4. AI Agent 在项目中的作用

AI Agent 在这个项目中主要承担了“开发助手”的角色。它可以帮助我把课程 PDF 和作业提示词转化成更具体的页面结构，也能快速生成初版代码，减少重复劳动。

具体来说，AI 主要帮助完成了这些工作：

  • 整理页面结构和功能模块；
  • 生成 HTML、CSS 和 JavaScript 初稿；
  • 优化卡片、按钮、时间线等视觉样式；
  • 补充响应式布局和滚动出现动画；
  • 检查博客入口、返回首页等链接是否合理。

但我也意识到，AI 生成的内容不能直接等于最终作品。网站内容和最终修改仍然需要自己检查，包括文字是否符合自己的表达、页面是否真的清楚、博客是否有个人理解。AI 更像是提高效率的工具，而不是替代思考的工具。

5. 我的思考

这次作业让我发现，AI Agent 很适合帮助完成重复性和结构化的开发任务。例如搭建页面框架、补充基础样式、生成响应式布局，这些工作如果完全手写会比较耗时，而 AI 可以快速给出一个可运行版本。

但是，真正决定网站质量的仍然是人的审美、判断和修改能力。尤其是个人主页这种项目，页面是否符合自己的风格、博客内容是否真实、有无个人理解，都需要自己不断调整。AI 可以把我从空白页面带到初版作品，但不能替我决定什么内容值得保留。

我觉得比较好的方式是：先让 AI 帮助搭出结构，再由自己检查逻辑、修改措辞、调整视觉细节。这样既能利用工具提高效率，也能保留个人表达。

6. 总结

通过这次算法基础大作业，我完成了一个包含个人介绍、项目展示、学习记录和技术博客入口的个人主页，也完成了第一篇关于 AI Agent 辅助开发的技术博客。

这次实践让我学会了如何用 AI Agent 辅助开发网页，理解了个人主页的基本结构，也对 AI 辅助编程有了更具体的认识。对我来说，最大的收获不是生成了多少代码，而是学会了如何把需求拆清楚、把学习过程表达出来，并在工具帮助下完成一个可以继续扩展的作品。
